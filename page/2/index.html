Go 学习记录（一）

package

package 声明

每个Go程序都是又包构成，程序执行的入门就是main包。
程序顶行声明包名。
package [packageName]

package 导入

有两种方式导入包。

  1. 一行一行导入
    Import “[packageName]”
  2. 一次性导入，既“分组导入”

Exported name 导出名

在 Go 中，如果一个名字以大写字母开头，那么它就是已导出的。
意思是，当我们写一个package时，如果声明的变量（函数）是以小写开头，那它就是私有的，当被别的package import时是无法访问这个变量（函数）的。

func 函数

在Go中，变量类型放在变量后面，返回类型放在函数名后面。

Go语言之所以用这样的语法，是因为当涉及到复杂的指针时，变量会变得难以理解以及不美观，例如
int (*fp)(int (*ff)(int x, int y), int b)
这是一个指向函数的指针，函数里的参数又是一个指向另一个函数的指针，让人头晕。
Go语言的类型后置更方便我们人类阅读。
例如，当我们用人类语言描述一样物品会使用这样的方式
Iphone：apple公司的智能手机。
Go语言只不过是把冒号和一些标点符号去掉而已。
x : 一个整数 ==> x int

类型省略

当函数的参数类型都一样时，可以省略除最后一个以外的类型。

多值返回

Go可以一次返回多个值，只需要在函数声明时写上全部返回类型就可以。（不可省略）
当接受多个返回时使用:= arg1, arg2:=func(args)

变量

var 变量声明

在Go中，用var 声明变量，准确的说是声明一个变量列表，看下面一段代码就可理解。

var a, b, c int 声明了一个列表，之后一次给列表的元素赋值。

Short variable 短变量

短变量是指func里的变量，也即是局部变量，局部变量当值已知时，可以不需要类型声明用:=声明

在声明一个变量而不指定其类型时（即使用不带类型的 := 语法或 var = 表达式语法），变量的类型由右值推导得出。
var a := 1

变量类型

Go的基本变量类型有

  • bool
  • string
  • int
  • float
  • complex
    除了这些之外，还有其它很多类型及其衍生，上述几种数字类型还有还有不同长度或者无符号的，例如int8, int6, uint32，这些以后再说。

JavaScript学习笔记（一）

数据类型

多行字符串不要用\换行，防止转义了空格难以发现。尽量使用闭合引号+，或者使用ES6的 ` 反引号。

Object

null表示空object，undefined表示空非object。

ES5以后object的最后一个key-value可以跟逗号，也就是说IE8以下不兼容。

object的key在声明是必定为string类型，若果不是的话自动转换为string。

当需要用key索引时，用object.[index]的方式索引时，index必须为字符串，因为中括号获取object的属性它并不会自动给你转成字符串。

当key为空字符串’’时，必须使用object[‘’]来获取。

object.atr的atr等价于object[‘atr’],所以在for..in中不应该用.atr变量value。

在用点访问对象的属性时，这个属性必须满足标识符的命名规则

初识HTTP

初识HTTP

HTTP全称“超文本传输协议”

超文本传输协议是一种用于分布式、协作式和超媒体信息系统的应用层协议。HTTP是万维网的数据通信的基础。 设计HTTP最初的目的是为了提供一种发布和接收HTML页面的方法。通过HTTP或者HTTPS协议请求的资源由统一资源标识符来标识。

通俗地讲，http就是互联网中网页间联系的纽带，是我们人为规定的协议，就像我们人类说的语言一样，不同国家的人（制订协议）规定自己国家的语言。在互联网中

请求

接下来学习中，将使用linux提供的cURL来模拟用户发送请求到服务器。

cURL是一个利用URL语法在命令行下工作的文件传输工具。它支持文件上传和下载。

curl -v -s https://www.baidu.com

通过curl的 -verbose可以输出这个请求的详细信息，再用-slient屏蔽一些多余的信息。

> GET / HTTP/1.1            
> Host: www.baidu.com
> User-Agent: curl/7.55.1
> Accept: */*
>

上面5行就是一个完整的请求header。

第一行：请求方式 请求路径 协议版本

第二行：域名

第三行：请求源

第四行：表示接受所有的返回

第五行：表示键值对分界线

我们还可以通过curl的各种参数构造不一样的请求。

请求的格式

1 动词 路径 协议/版本

2 Key1: value1

2 Key2: value2

2 Key3: value3

2 Content-Type: application/x-www-form-urlencoded

2 Host: www.baidu.com

2 User-Agent: curl/7.54.0

3

4 要上传的数据

  • 在标准的请求头中，至少要有三部分组成（1，2，3）最多不能超过四部分
  • 动词有 GET POST PUT PATCH DELETE HEAD OPTIONS 等
  • 路径（默认为\）包括查询串不包括锚点
